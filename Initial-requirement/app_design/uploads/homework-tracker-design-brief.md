# Design Brief: ระบบติดตามการบ้าน (Homework Tracker)

ส่งต่อให้ Claude Design เพื่อออกแบบ UI ครบทุกหน้า (Web App, responsive ทั้ง Desktop และ Mobile)

---

## Prompt สำหรับ Claude Design

```
ออกแบบ UI ของเว็บแอพ "ระบบติดตามการบ้าน" สำหรับครอบครัวที่มีลูกเรียนอยู่ในห้องเรียนเดียวกัน
ใช้ติดตามว่าการบ้านที่ครูสั่งแต่ละชิ้น ลูกทำหรือยัง ขาดอะไรบ้าง โดยแชร์ข้อมูลกลาง (มาสเตอร์การบ้าน)
ให้หลายครอบครัวใช้ร่วมกันได้ แต่แต่ละครอบครัวเห็นเฉพาะ progress ของตัวเอง

โทนการออกแบบ: สะอาด เป็นมิตร อ่านง่ายแม้ในมือถือจอเล็ก ใช้สีบอกสถานะชัดเจน
(เช่น แดง/ส้ม = ใกล้/เลยกำหนด, เขียว = เสร็จแล้ว, เทา = ยังไม่ทำ)
กลุ่มผู้ใช้มีทั้งเด็กและผู้ปกครอง ต้องใช้งานง่ายแม้ไม่ถนัดเทคโนโลยี

ออกแบบให้ครบทุกหน้าตามรายการและรายละเอียดในเอกสารแนบ (Functional/Non-functional spec)
โดยแบ่งเป็น 2 ระบบ: หน้าบ้าน (Parent/Child/Teacher) และ หลังบ้าน (Admin)
ทำเป็น responsive design รองรับทั้ง mobile และ desktop browser
```

แนบไฟล์นี้ทั้งฉบับ (หรือ copy ส่วน "Functional & Non-Functional Spec" ด้านล่าง) ไปพร้อมกับ prompt ข้างต้น

---

## รายการหน้าจอที่ต้องออกแบบ (Page List)

### A. หน้าบ้าน — ใช้ร่วมกันโดย Parent / Child / Teacher (ต่าง role เห็นต่างกันตาม permission)

1. **Login** — ปุ่ม Sign in with Google เท่านั้น
2. **Onboarding / สร้างหรือเข้าร่วมครอบครัว** — เลือก "สร้างครอบครัวใหม่" หรือ "เข้าร่วมด้วยลิงก์เชิญ"
3. **หน้าจัดการครอบครัว (Family Management)** — รายชื่อสมาชิกในครอบครัว (Parent/Child), ปุ่มเชิญสมาชิกใหม่ (เชิญผู้ปกครอง/เชิญลูกคนอื่น), สถานะคำเชิญ (pending/accepted)
4. **Dashboard หลัก (รายการการบ้าน)** — แสดงงาน active ทั้งหมด กรองตามชั้น+เทอม ของลูกแต่ละคนในครอบครัว มี filter/tab เลือกดูตามชื่อลูก แต่ละงานแสดง: วิชา, ครู, วันสั่ง, กำหนดส่ง, สถานะปัจจุบัน (สี), ปุ่มติ๊กเปลี่ยนสถานะ (ยังไม่ทำ/ทำแล้ว/ส่งแล้ว)
5. **หน้ารายละเอียดงาน (Assignment Detail)** — ข้อมูลเต็มของงานชิ้นนั้น + ปุ่มเปลี่ยนสถานะ + ปุ่ม "แจ้งแอดมิน" ถ้าข้อมูลผิด
6. **หน้าแจ้งปัญหา/คำขอแก้ไขมาสเตอร์ (Report Issue Form)** — ฟอร์มสั้นๆ เลือกงานที่เกี่ยวข้อง + พิมพ์รายละเอียด
7. **หน้า "คำขอของฉัน" (My Requests)** — list คำขอที่เคยแจ้งไป พร้อมสถานะ (pending/resolved/rejected) และข้อความตอบกลับจาก Admin
8. **หน้าสำหรับ Teacher (Class Overview / Inspector View)** — ตารางสรุป progress ของเด็กทุกคนในชั้นที่ตัวเองดูแล (read-only) แยกเห็นได้ว่าใครขาดงานอะไรบ้าง
9. **หน้าโปรไฟล์ผู้ใช้ (Profile)** — ดูข้อมูลตัวเอง, role, ชั้นเรียนที่เกี่ยวข้อง, ออกจากระบบ

### B. หลังบ้าน — Admin เท่านั้น

10. **Admin Dashboard ภาพรวม** — สรุปสถิติรวม (จำนวนครอบครัว, จำนวนงาน active, จำนวนคำขอค้าง, % completion รวม)
11. **จัดการมาสเตอร์การบ้าน (Assignment Master CRUD)** — ตาราง list งานทั้งหมด พร้อมปุ่ม เพิ่ม/แก้ไข/ลบ, ฟอร์มเพิ่ม-แก้ไข มีฟิลด์: วิชา, ครู, ชั้นเรียน, เทอม, วันสั่งงาน, กำหนดส่ง, active toggle
12. **ดู Progress รวมทุกครอบครัว** — ตาราง/การ์ด แสดงสถานะของทุกครอบครัวต่องานแต่ละชิ้น กรองตามชั้น/เทอม/สถานะ
13. **จัดการคำขอแก้ไข (Request Management)** — list คำขอจากผู้ใช้ทุก role พร้อมรายละเอียด, ปุ่ม approve/reject, ช่องพิมพ์ข้อความตอบกลับ
14. **จัดการ Audit Log** — ตาราง log การแก้ไขสถานะทั้งหมด (ใคร/เมื่อไหร่/เปลี่ยนอะไร/งานไหน) พร้อม filter ค้นหา
15. **จัดการครูประจำชั้น (Teacher Assignment)** — list ครู พร้อมชั้นเรียน+เทอมที่ดูแล, ปุ่มเพิ่ม/ถอดครูออกจากชั้น
16. **จัดการสมาชิก/ครอบครัวทั้งหมด (Family Overview สำหรับ Admin)** — list ครอบครัวทั้งหมดในระบบ พร้อมสมาชิกในแต่ละครอบครัว

## Functional & Non-Functional Spec (สำหรับแนบ)

### Roles
- **Admin**: full control มาสเตอร์การบ้าน, จัดการคำขอ, เห็น progress/log ทุกครอบครัว, มอบหมายครูเข้าชั้นเรียน
- **Parent**: แก้สถานะการบ้านของลูกทุกคนในครอบครัวตัวเอง, ส่งคำขอแก้ไขมาสเตอร์, เห็นเฉพาะครอบครัวตัวเอง
- **Child**: แก้สถานะการบ้านของตัวเองเท่านั้น (แก้ของพี่น้องไม่ได้), ส่งคำขอได้
- **Teacher**: ดู progress ของเด็กทุกคนในชั้นที่ตัวเองดูแลเท่านั้น (read-only, ดูได้หลายครอบครัว) ส่งคำขอได้ ไม่แก้ไขสถานะใครได้

### Authentication & Family
- Login ด้วย Google Sign-In เท่านั้น
- สร้างครอบครัวได้ก่อน แล้วเชิญสมาชิกผ่านลิงก์อีเมล (เชิญผู้ปกครองเพิ่ม หรือเชิญลูกคนอื่นเข้าครอบครัวเดียวกัน) — ผู้ถูกเชิญต้อง login ด้วย Gmail ที่ตรงกับอีเมลที่ถูกเชิญ ถือเป็นการยืนยันตัวตน
- ลูก 1 คน อยู่ได้แค่ 1 ครอบครัวเท่านั้น
- 1 ครอบครัวมีได้หลาย Parent และหลาย Child

### Master Assignment
- ฟิลด์: วิชา, ชื่อครู, ชั้นเรียน, เทอม, วันที่สั่งงาน, กำหนดส่ง, active flag (ผูกกับชั้นเรียน+เทอม)
- แก้ไขได้โดย Admin เท่านั้น ผู้ใช้อื่นแจ้งผ่านระบบคำขอ

### Progress Tracking
- สถานะ: ยังไม่ทำ / ทำแล้ว / ส่งแล้ว (ไม่มีแนบไฟล์/รูป)
- ผูกกับ (Child + Assignment) ไม่ใช่ผูกกับ Family ตรงๆ
- Data isolation: ครอบครัวหนึ่งห้ามเห็น/แก้ของอีกครอบครัวเด็ดขาด

### Audit Log
- บันทึกทุกการแก้ไขสถานะ: ใคร, เมื่อไหร่, เปลี่ยนจากอะไรเป็นอะไร, งานไหน, ของเด็กคนไหน

### Request/Issue Flow
- ทุก role (ยกเว้น Admin) ส่งคำขอแก้ไขมาสเตอร์ได้เท่ากัน
- คำขอมีสถานะ: pending → resolved/rejected พร้อมข้อความตอบกลับจาก Admin
- ผู้ส่งคำขอเห็นสถานะและคำตอบของตัวเองได้

### Notification
- แจ้งเตือนในหน้าเว็บแอพเท่านั้น (badge/สี) เมื่อใกล้ครบกำหนดส่งหรือเลยกำหนด ไม่มี noti ภายนอก (อีเมล/SMS/Line)

### Non-Functional
- Platform: Web App, responsive รองรับ Desktop และ Mobile browser
- Authentication: OAuth Google เท่านั้น
- Authorization: Role-based access control ตามที่ระบุข้างต้น
- Performance: โหลดหน้ารายการ active ได้เร็วแม้มีหลายครอบครัว/หลายชั้นเรียน
- Scope ปัจจุบัน: ออกแบบให้ใช้กับห้องเรียนเดียวก่อน แต่เผื่อโครงสร้างข้อมูลสำหรับขยายในอนาคต (ไม่ต้องสร้าง UI multi-school ตอนนี้)
